
# AleksOS BETA — NES консоль на базе ESP32

**AleksOS** — экспериментальный проект портативной игровой консоли на базе 
микроконтроллера **ESP32-2432S028** («Cheap Yellow Display»).  
Использует ядро NES-эмулятора **Nofrendo** с полной переработкой OSD-слоя 
под возможности ESP32.

---

## ⚠️ Важно: проект заточен под конкретное железо

Проект разработан специально под мою модифицированную плату.  
Работа на других платах, конфигурациях и ревизиях **не проверялась** 
и **не гарантируется**.

## Железо

| Компонент | Описание |
|-----------|---------|
| МК | ESP32 @ 240 МГц, 4 МБ Flash, 4 МБ PSRAM |
| Дисплей | ST7789 320×240 IPS, SPI DMA 80 МГц (LovyanGFX) |
| Звук | I2S внутренний ЦАП, GPIO26, 22050 Гц моно |
| Хранилище | MicroSD карта (SPI) |
| Контроллер | Raspberry Pi Pico через UART2 (GPIO22/27) |
| Вибро | 2× виброматора подключены к Pico (GP10, GP11) |
| Подсветка | Авто-яркость через фоторезистор (GPIO34) |
| Тачскрин | XPT2046 резистивный |

## Возможности

- NES эмуляция 50+ FPS
- Браузер ROM — сканирует SD карту, навигация кнопками и тачем
- Полный контроллер — A, B, Select, Start, крестовина
- Выход из игры — зажать **Select + Start** на 2 секунды
- Регулятор громкости — отдельно для меню и эмулятора
- Переназначение кнопок (ремаппинг)
- Несколько тем оформления
- Авто-яркость по датчику освещённости
- Вибро-отклик в меню и в игре
- WAV звуки меню с SD карты

## Модификация PSRAM

Плата модифицирована — добавлена внешняя микросхема PSRAM **APS6404L-3SQR-SN** 
(8 МБ QSPI). Для подключения используются выводы GPIO16/17, которые на оригинальной 
плате заняты RGB-светодиодом — светодиод удалён.

Автор идеи модификации: [hexeguitar](https://github.com/hexeguitar/ESP32_TFT_PIO/blob/main/README.md)

<img alt="cyd_PSRAM_mod" src="https://github.com/user-attachments/assets/13a14e45-822b-409a-b7bd-12c2a5f268f4"/>

**Подходящие микросхемы:**
- `APS6404L-3SQR-SN` — 8 МБ (рекомендуется)
- `APS1604M-3SQR-SN` — 2 МБ (минимум)

> ⚠️ **Обязательно версия `3SQR`** — модели без `3` рассчитаны на 1.8V, 
> в CYD это 3.3V, чип сгорит.

### ⚠️ Порядок установки PSRAM

На линиях RGB-светодиода при старте платы подаётся напряжение. 
Если припаять PSRAM **до** прошивки — напряжение попадёт прямо на 
выводы APS6404L и сожжёт микросхему.

**Правильный порядок:**
1. Сначала прошить плату
2. Затем припаять APS6404L-3SQR-SN
3. После этого включать питание

## Контроллер кнопок (Raspberry Pi Pico)

На плате ESP32-2432S028 практически нет свободных GPIO, поэтому 
в качестве контроллера кнопок используется **Raspberry Pi Pico**.  
Он сканирует кнопки и передаёт их состояние на ESP32 по UART.

**Подключение:**

| ESP32 | Pico | Назначение |

|-------|------|------------|

| IO22  | GP0 (TX) | Данные кнопок → ESP32 |

| IO27  | GP1 (RX) | Команды ESP32 → Pico |

| GND   | GND | Общая земля |

| 3.3V  | 3V3 | Питание Pico от ESP32 |


**Протокол:**
Pico → ESP32: [0xAA] [0x42] [кнопки] [~кнопки]

ESP32 → Pico: [0xAA] [cmd] [data] [cmd ^ data]

Биты кнопок: A=0x01 B=0x02 Sel=0x04 Start=0x08
↑=0x10 ↓=0x20 ←=0x40 →=0x80

### ⚠️ Прошивка Raspberry Pi Pico

**Перед прошивкой Pico — отключить все провода от ESP32 (GP0, GP1, 3V3, GND).**

Когда Pico подключается к ПК по USB, она начинает питать ESP32 через линию 3.3V. 
ESP32 не включена и тянет нагрузку всей платы через стабилизатор Pico — 
это ведёт к перегреву и выходу из строя.

**Правильный порядок:**
1. Отключить Pico от ESP32
2. Зажать BOOTSEL, подключить Pico к ПК по USB
3. Прошить Pico
4. Подключить обратно к ESP32

> При прошивке **ESP32** — отключать Pico **не нужно**.  
> В этом случае Pico питается от ESP32, а не наоборот.

# Клонировать и собрать
git clone https://github.com/YOUR_USERNAME/RetroESP_Console
cd RetroESP_Console
pio run

# Прошивка ESP32 (зажать BOOT, затем:)
pio run --target upload
