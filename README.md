AleksOS — это мой экспериментальный проект, основанный на эмуляторе NES **Nofrendo** 
и адаптированный под микроконтроллеры ESP32-2432S028 или дешовый жолтый дисплей Cheap Yellow Display

## ⚠️ Важное предупреждение по установке PSRAM

Плата была модифицирована: добавлена внешняя микросхема PSRAM 
APS6404L‑3SQR‑SN. Для связи с PSRAM используются выводы, которые на 
оригинальной плате были задействованы под RGB‑светодиод.

⚠️ ВАЖНО:
Если припаять PSRAM ДО прошивки платы — микросхема может сгореть.

Причина:
На линиях RGB‑светодиода при старте подаётся напряжение. 
Так как эти же линии используются для интерфейса PSRAM, 
напряжение попадает прямо на выводы APS6404L‑3SQR‑SN, что приводит 
к её повреждению или полному выходу из строя.

Правильный порядок установки:
1. Сначала прошить плату.
2. Затем припаять микросхему APS6404L‑3SQR‑SN.
3. После этого включать питание.


## Используемая PSRAM

Проект рассчитан на работу с внешней микросхемой PSRAM **APS6404L‑3SQR‑SN** 
(8 MB QSPI PSRAM). Прошивка и вся логика работы памяти адаптированы 
именно под эту модель.

Использование других вариантов PSRAM не гарантируется.

## Контроллер кнопок на Raspberry Pi Pico

В проекте используется отдельный микроконтроллер **Raspberry Pi Pico** 
в качестве контроллера кнопок. Он передаёт состояние кнопок на ESP32-2432S028 
по UART, так как на плате ESP32-2432S028 (Cheap Yellow Display) 
практически нет свободных GPIO.

Raspberry Pi Pico сканирует кнопки и отправляет их состояние в виде байта-маски.

## Подключение Raspberry Pi Pico

Подключение выполнено следующим образом:


ESP32 IO22  ←  Pico GP0 (Serial1 TX)   [RX]

ESP32 IO27  →  Pico GP1 (Serial1 RX)   [TX]

ESP32 GND   ─  Pico GND

ESP32 3.3V  →  Pico 3V3

## ⚠️ Важное предупреждение по прошивке Raspberry Pi Pico

Перед прошивкой Raspberry Pi Pico необходимо ОБЯЗАТЕЛЬНО 
отсоединить все провода, соединяющие Raspberry Pi Pico с ESP32-2432S028 
(GP0, GP1, 3V3, GND).

Причина:
Когда Raspberry Pi Pico подключается к компьютеру по USB, она получает питание 
от USB-порта и одновременно пытается ПИТАТЬ ESP32-2432S028 через линию 3.3V. 
ESP32-2432S028 в этот момент не включена, и Raspberry Pi Pico начинает тянуть на себя 
нагрузку всей ведущей платы.

Это вызывает сильную просадку напряжения, перегрев и может привести 
к выходу из строя стабилизатора питания на Raspberry Pi Pico.

То есть Raspberry Pi Pico начинает работать как «блок питания» для ESP32-2432S028, 
что недопустимо и опасно для Raspberry Pi Pico.

Правильный порядок:

1. Полностью отключить Pico от ESP32 (GP0, GP1, 3V3, GND).

2. Зажать BOOTSEL и Подключить Pico к ПК через USB.

3. Прошить Pico.

4. После прошивки снова подключить Pico к ESP32.

## Прошивка ESP32-2432S028

При прошивке ESP32-2432S028 отключать Raspberry Pi Pico НЕ нужно.

Причина:
Когда прошивается ESP32-2432S028, питание подаётся на ESP32-2432S028, и Raspberry Pi Pico получает 
питание ОТ ESP32-2432S028, а не наоборот. Raspberry Pi Pico не пытается питать ведущую плату, 
поэтому перегрузки стабилизатора не происходит.

То есть Raspberry Pi Pico безопасно питается от ESP32-2432S028, и никакой опасности нет.
